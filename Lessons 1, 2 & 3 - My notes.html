Lesson 1. The Basics of the Web and HTML

How the Web works

The web is a number of computers that communicate with each other. Computers send HTTP Requests to a sever. The server then finds the appropriate HTML documents and sends it back to the the computer that asked for it. A web browser interprets the page and displays it accordingly.

As such, the World Wide Web is a collection of HTML documents. This is the basis for almost every webpage. Hyperlinks are what makes the makes the web "web like".

Anything can exist on the web, from plain text, html, music, images, videos, etc.

HTML

HTML stands for Hypertext Markup Language. It is basically the heart of the web, as the majority of the content on the web are in HTML form.

HTML is made up of text content (what you see), markup (what it looks like) and references to other documents, such as images, videos, links to other pages.

In HTML plain text is plain text. HTML Markup is made up of things called tags. For example:

  • <b>anything</b> == bold
  • <em>anything</em> == italics

If you don't use a closing tag after the opening tag, everything after the tag will be considered part of it.

HTML Attributes

Composition:

  • <tag attr="value">contents</tag>

Examples of tags that use an attribute:

Anchor tag - <a>
  • <a href="www.reddit.com">Reddit</a>
Image tag - <img>
  • <img src="url" alt="text"> ---> src = source
  • < img src="" alt=""> ---> void tag = closes without tag because there is no content

Whitespace:

To force space you need to use the void tag <br> = break = line breaks.You put it after the word you want the line to break at. If not html puts everything in one line. Can also use the <p> tag. Is not a void tag, because it has content.

Content can be either inline or block

  • "br" - inline
  • "p" - block - makes an invisible block (can actually have height and width)

Container elements:

Elements that do nothing but contain stuff inside of them
  • <span>text</span> ---> inline
  • <div>text</div> ---> block

with CSS:

  • <span class="foo">text</span>
  • <div class="bar">text</div>
  • Inline Elements so far = a, span, br, img, strong
  • Block Elements so far = div, p, form

Structure of HTML document

  • <!DOCTYPE HTML> - doctype
  • < html>
  • < head> - meta-data (javascript, css)
  • <title>Title!</title>
  • </head>
  • <body> - body - contents
  • <b>Content</b>
  • </body>
  • </html>

Lesson 2. Creating a Structured Document with HTML

Developer Tools in the Browser

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

The "tree like" structure of HTML

The "tree like" structure comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes.

The relationship between indented HTML and boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.

Text Editors for Programming

When writing code, programmers use special text editors. These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.

Lesson 3. Adding CSS Style to HTML Structure

CSS

CSS (Cascading Style Sheets) give programmers the ability to control the style of HTML elements by giving similar HTML elements the same class name and then specifying the style that should apply to all that class.

Avoiding Repetition

There are several reasons for which programmers should avoid repetition. First, it helps them to avoid errors. While constantly copy and pasting code, or rewriting the same code with minor differences many times, the chances of committing errors increase. This is why they should diligently make the changes everywhere the repeated code appears.

Second, is aiming for consistency. By writing easy to reproduce code and streamlining several style rules, the programmer can easily alter any of the pieces and affect the whole documents instead of having to go line after line looking for where to effect changes or not.
